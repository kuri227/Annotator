# Annotator

Annotatorは、AIモデル開発に必要な分類用教師データを効率よく作成するWindows向けデスクトップアプリケーションです。

大量のデータを1件ずつ確認してラベルを記録し、学習用フォルダへ整理する作業には多くの時間がかかります。本プロジェクトは、画像と音声の確認、ラベル選択、進捗管理、教師データの整理を1つの画面と共通した操作方法にまとめ、アノテーション作業の高速化と管理負担の軽減を目指しています。

現在は、画像および音声にクラスを付与する「分類アノテーション」に対応しています。1ファイルに1クラスを付ける単一ラベル分類と、犬と猫の音が同時に含まれる音声などへ複数クラスを付けるマルチラベル分類の両方を扱えます。

## このアプリでできること

### 画像分類

- JPG / JPEG / PNG / BMP / WebP画像を自動検出
- 画像を画面サイズに合わせ、縦横比を維持して表示
- 表示中の画像へ1つまたは複数のラベルを付与
- 前後の画像へボタンまたは矢印キーで移動
- 数字キーで最大10クラスをすばやく選択・解除
- 現在のファイル名、表示位置、選択ラベル、完了件数を常時表示
- 分類済み・未分類画像の一覧確認と対象画像への直接移動

### 音声分類

- WAV / MP3 / FLAC / OGG / M4A / AAC音声を自動検出・再生
- PCM WAVの波形を表示し、音の有無や発生位置を視覚的に確認
- 音声へ1つまたは複数のラベルを付与
- 再生・一時停止、シーク、5秒送り・戻し、速度・音量変更
- 波形クリックによる再生位置の移動
- ファイル切り替え時の自動再生
- 大容量音声の読み込み状態、経過秒数、取得可能な場合の進捗率を表示
- 読み込み中でも前後のファイルへ移動可能
- 現在のファイル名、表示位置、選択ラベル、完了件数を常時表示
- 分類済み・未分類音声の一覧確認と対象音声への直接移動

### データ管理と作業進捗

- `raw_data`以下のサブフォルダを含めて対象ファイルを検出
- `raw_data`からの相対パスで管理し、サブフォルダ間の同名ファイル衝突を防止
- CSVまたはJSON形式でアノテーションを保存
- ラベル済みファイルを`input_data`へコピーし、元のフォルダ階層を維持
- 全ファイル数、現在位置、分類済み件数、未分類件数を表示
- ファイル一覧を「すべて」「分類済み」「未分類」で絞り込み
- 一覧を20件単位で表示し、大規模データセットでも操作負荷を抑制
- 既存の単一ラベルCSV / JSONも読み込み可能

## 想定する利用の流れ

```text
未加工データをraw_dataへ配置
          ↓
画像分類または音声分類を選択
          ↓
ファイルを確認し、クラスを選択
          ↓
進捗一覧で未分類データを確認
          ↓
CSV / JSONとinput_dataへ保存
          ↓
AIモデルの学習・評価で利用
```

本アプリは元の`raw_data`を変更しません。保存時にアノテーション情報をCSV / JSONへ書き出し、ラベルを付けたファイルを`input_data`へコピーします。この構成により、原本、アノテーション結果、学習用入力を分けて管理できます。

## ダウンロードと起動

Python環境を用意せず利用する場合は、[Annotator.exe](dist/Annotator.exe)をダウンロードして起動してください。

- 対応OS: Windows 10 / 11（64bit）
- バージョン: 5.3.0
- インストール: 不要
- 実行ファイルは未署名のため、初回起動時にWindowsの警告が表示される場合があります。

ソースコードから起動する場合:

```powershell
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
.venv\Scripts\python.exe app.py
```

## プロジェクトの準備

アノテーション対象は、リポジトリとは別のプロジェクトフォルダで管理することを推奨します。

```text
my_annotation_project/
├─ raw_data/          # 元の画像・音声。サブフォルダも使用可能
├─ input_data/        # ラベル済みファイルの学習用コピー
└─ annotations/       # CSV / JSONのアノテーション情報
   ├─ image_labels.csv
   └─ audio_labels.csv
```

最初に必要なのは`raw_data`と、その中の画像または音声だけです。プロジェクトフォルダを選ぶと、アプリが`input_data`と`annotations`の推奨パスを設定します。保存時に存在しない出力フォルダは自動作成されます。

## 基本操作

1. `Annotator.exe`または`app.py`を起動します。
2. 「画像分類」または「音声分類」を選択します。
3. プロジェクトフォルダを選択します。
4. `raw_data`、`input_data`、アノテーション保存先を確認します。
5. クラス名をカンマ区切りで入力します。クラスは最大10個です。
6. 「アノテーションを開始」を押します。
7. 内容を確認し、数字キーまたはボタンで必要なラベルを選択します。
8. `Enter`で選択を確定し、次のファイルへ進みます。
9. 必要に応じて一覧画面から未分類ファイルへ移動します。
10. `Ctrl+S`でアノテーションと学習用ファイルを保存します。

ラベルボタンは選択状態が維持され、同じファイルに複数のボタンを選択できます。選択中のラベルをもう一度押すと解除されます。

## ショートカット

### 画像・音声共通

| キー | 動作 |
|---|---|
| `1`〜`9`, `0` | 対応するラベルの選択・解除 |
| `Enter` | 選択中のラベルを確定して次へ |
| `←` / `→` | 前後のファイルへ移動 |
| `S` | ラベルを変更せず次へ |
| `Ctrl+S` | アノテーションを保存 |

### 音声操作

| キー | 動作 |
|---|---|
| `Space` | 再生・一時停止 |
| `J` | 5秒戻す |
| `L` | 5秒進める |

## ファイル一覧と進捗確認

アノテーション画面の「ファイル一覧・未分類を確認」から、プロジェクト全体の状態を確認できます。

- 元データ内での通し番号
- 分類済み・未分類の状態
- 付与済みのすべてのラベル
- `raw_data`からの相対ファイルパス
- 分類済み件数、未分類件数、全件数

一覧は20件単位でページングされます。状態による絞り込みが可能で、行をダブルクリックすると、そのファイルをアノテーション画面で開きます。

## 出力されるアノテーション

### CSV

CSVは`filename,labels`の2列です。`filename`には`raw_data`からの相対パス、`labels`にはJSON配列を保存します。

```csv
filename,labels
animals/dog_and_cat.jpg,"[""Dog"", ""Cat""]"
audio/environment.wav,"[""Bird"", ""Wind""]"
```

### JSON

```json
{
  "animals/dog_and_cat.jpg": ["Dog", "Cat"],
  "audio/environment.wav": ["Bird", "Wind"]
}
```

日本語やカンマを含むラベルも保存できます。また、旧バージョンの`filename,label`形式CSVと、値が単一文字列のJSONは、1件のラベルとして読み込まれます。

## データ保護と安定性

- アノテーションは一時ファイルへ書き出した後に置換し、保存途中の破損を防止します。
- 元の`raw_data`は変更せず、学習用ファイルは`input_data`へコピーします。
- 個別ファイルのコピーに失敗しても、ラベル情報の保存結果は維持します。
- 未保存の変更がある状態で戻る、または終了すると確認画面を表示します。
- 波形解析をバックグラウンドで行い、大容量音声によるUI停止を防ぎます。
- 破損・非対応音声はエラーとして表示し、別のファイルへ移動できます。
- 予期しないUI例外を捕捉し、回復可能な場合はアプリを継続します。
- 診断ログは`%LOCALAPPDATA%\Annotator\logs\annotator.log`へ最大1MB、2世代で保存します。
- 通常の操作履歴、ファイル内容、アノテーション内容は診断ログへ記録しません。

## 現在の対応範囲

| データ | 対応形式 | アノテーション方法 |
|---|---|---|
| 画像 | JPG, JPEG, PNG, BMP, WebP | 単一ラベル分類、マルチラベル分類 |
| 音声 | WAV, MP3, FLAC, OGG, M4A, AAC | 単一ラベル分類、マルチラベル分類 |

現時点では分類アノテーションが対象です。バウンディングボックス、セグメンテーション、音声区間指定、動画アノテーションにはまだ対応していません。

## 開発、テスト、EXE作成

Python 3.10〜3.12を推奨します。

```powershell
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements-dev.txt
.venv\Scripts\python.exe -m unittest discover -v
```

テストでは一時的なCSV、PNG、WAVを実際に生成し、画像・音声表示、単一・マルチラベル保存、旧形式互換、一覧表示、音声切替、異常ファイル、原子的保存を検証します。

Windows用EXEは次のコマンドで作成します。

```powershell
.\build.ps1
```

スクリプトは全テストを実行した後、PyInstallerでコンソール非表示の`dist\Annotator.exe`を生成します。
